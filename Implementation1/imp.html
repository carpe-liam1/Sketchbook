Project 2 - Implementation 1

Implementation Details

So far I’ve implemented the overall layout of my interface prepositions from figma, specifically for the web layout (I still haven’t implemented a mobile layout). Currently, the site now contains labels for all the different controls along with an explanation/instruction manual in the bottom left corner of the site. These all are provided to give the user better control along with seamless function of the site’s tools. I found that one of the most difficult parts, surprisingly, was the css code since I had to understand how to integrate the js code elements along with functional tools like that of the slider. I specifically had to experiment with the layout features multiple times before I had a good understanding of how big certain features like the slider and canvas were. When it comes to the navigational elements, I think the html assumes that the user is coming specifically to use the manipulation tool and not explore or analyze the production and manufacturing behind it. I think one thing I need to add in is features to facilitate going to related sketches and progress reports so that they can be viewed alongside the sketch itself. This would likely come in the form or links which take the user to previous versions of the type tool. The overall layout automatically creates a very simple hierarchy between the navigation, the explanatory features, tools, and the canvas itself. I tried to label as much as I could without letting the site feel cluttered. I found that the figma logic which I implement, whose goal was specifically to make the site more digestible and understandable, holds up fairly well. Every tool is explicitly marked and explained, making them all distinguishable. I think another thing to add is to change the features of the undo tool to more of a revert feature which takes the shape back through its previous forms, as opposed to going just straight back to the original root text. I think this is where the overall composition breaks since it is kind of limiting in terms of a beginning to ending binary, instead of exploring the type change in between.
