Reading and Response: Shneiderman's 8 Golden Rules

Which rule feels most absent from your micro-project page?


The rule most absent from my micro-project is absolutely the reversibility factor, specifically the element of permitting an easy reversal of actions. With the way that revert button functions, the reversal does not actually mimic the distortion which occurred before which is really unfortunate since it doesn’t facilitate the release of anxiety which Shneiderman outlines is created through easy reversibility.



Which rule would most improve your sketchbook interface?


I think that adding instructions/suggestions would improve my interface the most since It would help users both understand how to use my tool as well as how it can be utilized for different visuals.



Which rule is hardest to implement honestly?


I think the reversibility rule is the hardest one to implement since it would require the most amount of new code. The idea of making a function which could go back to certain states would be something new to me to develop and also require a way to record all previous states which seems data heavy.



Did any rule contradict how you designed your system originally?


Not fully, though I do think that the idea of informative feedback was something which I viewed as not necessary to the web page. This is because the goal of the web page is for personal exploration and interpretation, and feedback would be trying to direct these actions in some way if it were just to explain actions.
